# Changelog

## 0.3.9

- **Speaking speed:** User-facing rate changes are **off** for now — **`speaking_rate`** stays **1.0** (config / `--rate` / persisted state ignored). **Ctrl+Alt+Plus/Minus** are no longer hooked. Code paths for live-rate playback remain for a future restore.
- **XTTS v1.1 default speaker:** Checkpoints like **`xtts_v1.1`** have no built-in named speakers (fixes `'NoneType' object has no attribute 'speakers'`). With no `xtts_speaker_wav`, Narrator **creates a cached clone WAV** via the default **Piper** voice under `%LOCALAPPDATA%\\narrator\\xtts\\` — [`narrator/tts_xtts.py`](narrator/tts_xtts.py). Use **`xtts_v2`** for Coqui names like Ana Florence without cloning.
- **XTTS + Ollama UX:** `run_narrator.bat` sets **`COQUI_TOS_AGREED=1`** when unset so Coqui does not block on stdin; [`narrator/tts_xtts.py`](narrator/tts_xtts.py) also sets it before load. Default **`speak_text_llm_bundle_chunks`** is **`1`** (one LLM call per TTS segment — reliable for **`llama3.2:1b`**); use **`2`–`4`** with larger models that return `<<<CHUNK>>>` markers. Bundle parse failures log **INFO** only (no duplicate **WARNING**). Delimiter typo normalization in [`narrator/speak_text_llm.py`](narrator/speak_text_llm.py).
- **Lighter neural defaults:** Default **XTTS** model **`tts_models/multilingual/multi-dataset/xtts_v1.1`** (smaller than `xtts_v2`; override `xtts_model` for v2). Default speak-text **Ollama** model **`llama3.2:1b`** when unset (was `llama3.2`). LLM system prompt now includes an explicit **CONSTRAINTS** block (segment caps, bundle shape, `speak_engine`, mode) before **RULES** — [`narrator/speak_text_llm.py`](narrator/speak_text_llm.py), [`narrator/tts_xtts.py`](narrator/tts_xtts.py), [`narrator/settings.py`](narrator/settings.py).
- **Smoother XTTS / multi-segment playback:** Chunk-context trim applies a short **head fade-in** after removing overlap ([`narrator/audio_pcm.py`](narrator/audio_pcm.py) `wav_fade_in_head_ms`, [`narrator/speech.py`](narrator/speech.py)). **XTTS**: internal micro-segment join crossfade **12 → 24 ms** ([`narrator/tts_xtts.py`](narrator/tts_xtts.py)); **engine** preset edge/crossfade **~10 / ~38 ms** ([`narrator/segment_transitions.py`](narrator/segment_transitions.py)). Default **`speak_prefetch_depth`** **3 → 4** to reduce boundary gaps when synthesis lags realtime ([`narrator/settings.py`](narrator/settings.py), [`narrator/worker.py`](narrator/worker.py)).
- **Paper front matter (default on):** `speak_start_at_abstract` defaults to **`true`** — skip title, author lists, and lines before **Abstract** (more PDF layouts: heading on its own line, `Abstract:` / em dash, same-line `Abstract The …`). Affiliation / contact paragraphs (university, **community**, `{ids}@host`) collapse to **see names**; `{user1, user2}@domain` removed in [`narrator/speak_preprocess.py`](narrator/speak_preprocess.py). Set **`speak_start_at_abstract = false`** when you need the full capture read aloud.
- **XTTS long documents — faster playback:** Default chunk-context trim is **`fixed_ms`** (no extra context-only XTTS synthesis per segment; **`duration_probe`** remains available for finer trim). **`XTTS_MAX_CHARS_PER_SEGMENT`** raised **220 → 242** (still under Coqui’s ~250-char en warning) for fewer segments. **`speak_text_llm`**: lenient parsing when the LLM omits **`<<<END>>>`** markers; **`temperature` 0.1** for stabler bundle format; strips leading markdown fences. See [`narrator/speech.py`](narrator/speech.py), [`narrator/speak_chunking.py`](narrator/speak_chunking.py), [`narrator/speak_text_llm.py`](narrator/speak_text_llm.py).
- **Quieter setup/run console:** [`setup.bat`](setup.bat), [`scripts/bootstrap_install.py`](scripts/bootstrap_install.py), [`scripts/ensure_cuda_torch.py`](scripts/ensure_cuda_torch.py), prefetch/Ollama/verify helpers, and [`scripts/ensure_ollama_running.bat`](scripts/ensure_ollama_running.bat) default to **short** output; set **`NARRATOR_SETUP_VERBOSE=1`** for the previous detail ([`scripts/setup_terminal.py`](scripts/setup_terminal.py)). [`docs/SETUP.md`](docs/SETUP.md) notes the flag.
- **Speak-text LLM — default RULES file:** Packaged [`narrator/default_speak_text_llm_rules.txt`](narrator/default_speak_text_llm_rules.txt) is prepended to the system `RULES` block (toggle `speak_text_llm_builtin_rules`, env `NARRATOR_SPEAK_TEXT_LLM_BUILTIN_RULES`). Shorter structural prompts in [`narrator/speak_text_llm.py`](narrator/speak_text_llm.py); wheel includes `package-data` for the `.txt`. See [`docs/SPEAK_TEXT_LLM.md`](docs/SPEAK_TEXT_LLM.md).
- **Local LLM text readying:** OpenAI-compatible `POST /v1/chat/completions` per chunk (`speak_text_llm_*`). **Piper** and **XTTS** always enable the LLM pass with default model **`llama3.2:1b`** when unset; **WinRT** stays opt-in. [`scripts/setup_ollama_speak_llm.py`](scripts/setup_ollama_speak_llm.py) (winget **Ollama** + `ollama pull`) runs from [`bootstrap_install.py`](scripts/bootstrap_install.py) neural post-install; skip with **`NARRATOR_SKIP_OLLAMA_SETUP`**. Modes `heuristic_then_llm` / `llm_primary`, rules file + inline rules — [`narrator/speak_text_llm.py`](narrator/speak_text_llm.py), [`docs/SPEAK_TEXT_LLM.md`](docs/SPEAK_TEXT_LLM.md). **Synth pipeline:** `speak_synth_max_ahead`, parallel `speak_synth_worker_threads` (non-streaming builds), optional `speak_keep_wav_in_memory`; [`narrator/worker.py`](narrator/worker.py). Playback accepts WAV **bytes** or **Path** ([`narrator/wav_play_win32.py`](narrator/wav_play_win32.py)).
- **Two-step workflow:** **[`run.bat`](run.bat)** is the short launcher for **[`run_narrator.bat`](run_narrator.bat)**; both run **[`scripts/ensure_ollama_running.bat`](scripts/ensure_ollama_running.bat)** before **`ensure_cuda_torch.py`** so **Ollama** serves Piper/XTTS text LLM when installed. **[`run_narrator_tray.bat`](run_narrator_tray.bat)** does the same. README / **[`docs/SETUP.md`](docs/SETUP.md)** document **setup.bat** → **run.bat** as the default path.
- **Faster time-to-first-audio (LLM):** [`narrator/worker.py`](narrator/worker.py) LLM-readies the **first bundle** before building the first synth/play work items; **later bundles** run in a **background thread** while TTS/playback can start. Prefetch supports **growing** `work_items` when `prefetch_dynamic`.
- **Speak preprocess (papers / figures):** [`narrator/speak_preprocess.py`](narrator/speak_preprocess.py) removes **multi-reference** parentheticals `(A et al., 2024; B et al., 2025a; …)`, **full figure/table caption lines**, standalone **legend** lines like `LLMs (Blue)` and **timeline year** rows; **collapses** comma-separated **author-style lists** longer than **`speak_long_name_list_max`** (default **4**) to the phrase **see names** while **keeping** affiliation blocks with `{ids}@domain` or university + `@`. Settings: `speak_strip_figure_legend_lines`, `speak_collapse_long_name_lists`, `speak_long_name_list_max`; env `NARRATOR_SPEAK_*`.
- **LLM bundled chunks:** [`narrator/speak_text_llm.py`](narrator/speak_text_llm.py) optional multi-chunk bundles via **`speak_text_llm_bundle_chunks`** (default **`1`**, no markers). Settings **`speak_text_llm_bundle_max_chars`** (default **16000**). Fallback: per-chunk if bundle markers are missing.
- **Speak preprocess — math:** Stricter `speak_exclude_math`: more AMS environments, MathML, `tikzpicture` blocks, repeated passes for nested display/inline math, and Unicode mathematical operator/symbol ranges so formulas are not read aloud ([`narrator/speak_preprocess.py`](narrator/speak_preprocess.py)).
- **Streaming speak preprocess:** For large captures, only an initial paragraph-bounded raw bundle is preprocessed before the first synthesis; the remainder preprocesses in parallel (`speak_preprocess_streaming`, `speak_preprocess_initial_chunks`). See [`narrator/worker.py`](narrator/worker.py) and [`narrator/speak_chunking.py`](narrator/speak_chunking.py) `split_raw_for_streaming_preprocess`.
- **Voice cleaning (playback):** Optional `speak_voice_clean_enabled` + `speak_voice_clean_highpass_hz` — gentle **high-pass** (SciPy `sosfiltfilt`) before peak normalize; documented in [`docs/TTS_VOICE_CLEANING.md`](docs/TTS_VOICE_CLEANING.md) with broader industry techniques (LUFS, de-essing, limits of noise reduction).
- **Chunk context (multi-segment speak):** `speak_chunk_context_enabled` defaults to **`true`** — prepend the tail of the previous chunk to the next synthesis for smoother prosody on **WinRT, Piper, and XTTS**; [`narrator/speech.py`](narrator/speech.py) **`apply_chunk_context_trim`** removes the overlapped audio (`duration_probe` = extra context-only synthesis to measure trim length, or `fixed_ms`). See [`docs/TTS_PLAYBACK_ROADMAP.md`](docs/TTS_PLAYBACK_ROADMAP.md).
- **Speak preprocess (papers / tech):** Optional stripping of **arXiv** header lines, **TOC** dot-leader lines, and **contents-page** paragraphs; **`speak_start_at_abstract`** skips title/authors and starts at the abstract; **`speak_expand_tech_abbreviations`** spells acronyms like LLM/CLI for clearer TTS ([`narrator/speak_preprocess.py`](narrator/speak_preprocess.py)). Config keys and `NARRATOR_SPEAK_*` env vars in [`narrator/settings_schema.md`](narrator/settings_schema.md).
- **XTTS long PDFs:** Per-segment cap **~242** chars (under Coqui **en** ~**250**-char warning; longer input risks truncation and CUDA indexing errors). **PDF extraction:** strip **U+FFFC** (object replacement) in [`narrator/speak_preprocess.py`](narrator/speak_preprocess.py). **Speak preprocess:** normalize Unicode hyphens / soft hyphen (U+00AD); **chunking:** merge a **short final segment** into the previous chunk when it fits (less robotic tail). See [`docs/GLITCH_REGRESSION.md`](docs/GLITCH_REGRESSION.md) for stutter vs hyphen artifacts.

- **Setup hardening (Windows):** [`scripts/hw_detect.py`](scripts/hw_detect.py) now reports CPU (PowerShell / `platform`), logical cores, total RAM (Win32 API), free disk on the repo drive (warns if low), ARM64 note, and a basic **VC++** check (`vcruntime140.dll` in System32). [`setup.bat`](setup.bat) rejects **Python 3.15+** before creating the venv (Coqui still expects **Python below 3.15**). **Bootstrap** documents a **no-uninstall** policy; only `pip install` runs in the project venv.
- **Setup.bat / neural-gpu:** Console-safe ASCII dashes (no mojibake in `cmd.exe`). Pip upgrades **`setuptools`** with **`>=61,<82`** to match **PyTorch**’s constraint and avoid resolver noise. [`install_torch_cuda`](scripts/bootstrap_install.py) uses **`--force-reinstall`** so **`+cpu`** PyTorch is replaced by CUDA wheels from the configured index.
- **CUDA PyTorch / isolated venv:** [`scripts/pytorch_cuda_wheels.py`](scripts/pytorch_cuda_wheels.py) centralizes CUDA **`torch`/`torchaudio`** installs **into the active venv only**: **`pip install --upgrade`** from the CUDA index first; **`--force-reinstall`** only if CUDA still does not load. [`ensure_cuda_torch.py`](scripts/ensure_cuda_torch.py) and [`bootstrap_install.py`](scripts/bootstrap_install.py) use it. **`NARRATOR_CUDA_FORCE_REINSTALL=1`** skips the gentle step. [`docs/SETUP.md`](docs/SETUP.md) documents isolated **`.venv`** and optional offline **`pip download`** flow.
- **XTTS:** [`narrator/tts_xtts.py`](narrator/tts_xtts.py) calls **`tts_to_file` once** (no `try`/`except TypeError` fallback that could run Coqui synthesis twice). Optional **`speed=1.0`** only if the installed Coqui API exposes that parameter. **`split_sentences`** is **False** for text ≤2500 chars (faster on CPU); **True** above that (lower peak VRAM). Logs a **warning** when loading the model on **CPU only** (PyTorch without CUDA — GPU stays idle in Task Manager).
- **Idempotent prefetch:** [`scripts/prefetch_utils.py`](scripts/prefetch_utils.py) skips **XTTS** / **Piper** prefetch when Hugging Face / Piper caches already look ready (no scanning of arbitrary Downloads folders). Override with **`--prefetch-always`** on prefetch scripts or **`NARRATOR_FORCE_PREFETCH=1`**. **`minimal`** profile skips neural prefetch (Coqui not installed). CI runs **`pytest tests/`** (includes [`tests/test_prefetch_utils.py`](tests/test_prefetch_utils.py)); see [`.github/workflows/ci.yml`](.github/workflows/ci.yml).

### Internals (playback / live-rate stack; not user-facing in 0.3.9)

**Ctrl+Alt+Plus/Minus** are **not** hooked this release (see **Speaking speed** bullet at top). The following remains in the codebase for smoother segment joins and for a **future** restore of in-play rate changes; configuration still affects internal `waveOut` handoffs if you fork and re-enable hotkeys.

- **Live-rate engines (dormant):** Default **`live_rate_in_play_engine = "wsola"`** (audiotsm); alternatives **`phase_vocoder`** (librosa), **`resample`**. **`live_rate_in_play_use_phase_vocoder`** in old configs maps to **`phase_vocoder`**. Next-utterance-only: **`live_rate_defer_during_playback`**. Helpers in [`narrator/wav_speaking_rate.py`](narrator/wav_speaking_rate.py).
- **Live-rate handoff:** ``waveOutOpen`` after a speed change is retried with backoff; an empty remainder no longer forces a bogus reopen. Tuning: ``post_waveout_close_drain_s`` / ``NARRATOR_POST_WAVEOUT_CLOSE_DRAIN_S``.
- **Live-rate quality:** Speed-change handoff waits for the current ``waveOut`` buffer where applicable; PCM chunks capped at **64 KiB**; WSOLA uses **2048-sample** frames.
- **Playback smoothness:** **Edge fades** — ``pcm_edge_fade_ms``; **live-rate settle** — ``live_rate_settle_ms``; **extreme** in-play ratio — ``live_rate_extreme_ratio_threshold`` and resample fallback. Env: ``NARRATOR_PCM_EDGE_FADE_MS``, ``NARRATOR_LIVE_RATE_SETTLE_MS``, ``NARRATOR_LIVE_RATE_EXTREME_RATIO``.
- **Segment transitions:** ``segment_transition_preset = engine`` (WinRT / Piper / XTTS presets); see [`narrator/segment_transitions.py`](narrator/segment_transitions.py). Env: ``NARRATOR_SEGMENT_TRANSITION_PRESET``.
- **Advanced playback:** **Remainder re-synthesis** on live rate change (``live_rate_resynth_remainder``); **PortAudio** optional (``audio_output_backend = "sounddevice"``). **Telemetry** — ``NARRATOR_AUDIO_STATS=1`` and ``narrator.playback_telemetry.snapshot``. **API** — ``narrator.playback_result.PlayWavResult`` from ``play_wav_interruptible``. Docs: [`docs/GLITCH_REGRESSION.md`](docs/GLITCH_REGRESSION.md). Last user-facing live-rate notes: **0.3.8** below.

- **Setup (Windows):** [`setup.bat`](setup.bat) calls [`scripts/bootstrap_install.py`](scripts/bootstrap_install.py) with **`--auto`**, using [`scripts/hw_detect.py`](scripts/hw_detect.py) (`nvidia-smi` when available) to pick **`neural-gpu`** (install neural extras, then **CUDA** `torch`/`torchaudio` from the **`cu124`** wheel index) vs **`neural-cpu`**. Overrides: **`NARRATOR_SETUP_PROFILE`** (`minimal` / `neural-cpu` / `neural-gpu`), **`NARRATOR_TORCH_CUDA_INDEX`**, bootstrap flags **`--dry-run`**, **`--skip-prefetch`**, **`--force-cuda`**. Docs: [`docs/SETUP.md`](docs/SETUP.md), [`docs/REPO_LAYOUT.md`](docs/REPO_LAYOUT.md). [`CONTRIBUTING.md`](CONTRIBUTING.md) for contributors. [`setup.sh`](setup.sh) only prints that the runtime targets Windows.

- **Audio loopback self-test (optional):** [`pip install -e ".[audio-test]"`](pyproject.toml) adds **`sounddevice`**. [`run_audio_selftest.bat`](run_audio_selftest.bat) records the default mic to **`audio_selftest_logs/`**, then runs [`scripts/analyze_audio_recordings.py`](scripts/analyze_audio_recordings.py) (echo / overlap heuristics from WAV). Optional **`--openai`** + **`OPENAI_API_KEY`** + **`pip install openai`** sends **numeric metrics only** to the API for a short interpretation. See [`docs/AUDIO_SELFTEST.md`](docs/AUDIO_SELFTEST.md).

## 0.3.8

- **Standard speak profile** is the default: all **`speak_exclude_*`** remain **`true`**, and prosody is **paragraph-only** — pauses between blank-line-separated blocks, not after every single newline (`speak_pause_between_lines = false`). Opt into line-level pauses with `speak_pause_between_lines = true` or **`--speak-pause-between-lines`**. WinRT SSML uses paragraph breaks only unless line pauses are enabled.
- **Piper speaking rate:** Tempo is applied with Piper **`length_scale`** during synthesis instead of librosa **whole-file** time stretch, so changing rate no longer adds a persistent “chorus” on every following utterance. WinRT / XTTS still use ``wav_speaking_rate`` librosa when ``speaking_rate ≠ 1``.

- **Live rate:** Optional **`live_rate_defer_during_playback`** / **`NARRATOR_LIVE_RATE_DEFER=1`** — rate hotkeys update the **next** utterance only (no librosa stretch during playback), avoiding phase-vocoder chorus artifacts.

- **Debug:** `NARRATOR_DEBUG_AUDIO=1` logs `playback_gate`, `waveOut` open/close/reset/write, PCM stats, librosa live stretch, worker/speech play boundaries; see **`docs/DEBUG_MULTIPLE_VOICES.md`**.

- **Live speaking-rate playback (Ctrl+Alt+Plus/Minus):** Default **chunk-boundary resume** (skip the rest of the current buffer) so **`waveOutGetPosition` is not the sole cut point**—it often lags the DAC and caused stacked/echo speech. Optional **sample-accurate** seek via `live_rate_safe_chunk_discard = false` or **`NARRATOR_LIVE_RATE_ACCURATE_SEEK=1`** (uses position + higher default slack/drain). Safer fallbacks when position fails or is out of range; **8-bit** mono PCM tails converted for live stretch. Tunables TOML / env (`NARRATOR_LIVE_RATE_*`). **`NARRATOR_DEBUG_LIVE_RATE`** logs `raw_waveout_cb`, offset, `accurate_seek`. Unit tests: `tests/test_wav_play_win32.py`. Docs: README, ARCHITECTURE §4.4.

## 0.3.7

- **Speak prosody** ([`narrator/speak_prosody.py`](narrator/speak_prosody.py)): optional pauses between **lines** (single newline) and **paragraphs** (blank line). **Piper / XTTS** insert `, ` between lines and `. ` between paragraph blocks. **WinRT** uses SSML ``<break time="…ms"/>`` (defaults **320 ms** line / **520 ms** paragraph; configurable). ## 0.3.6

- **Speak preprocessing** expanded ([`narrator/speak_preprocess.py`](narrator/speak_preprocess.py)): optional stripping of **markup** (fenced code, HTML tags/entities, markdown headings / bold markers / list bullets / horizontal rules; markdown images and `[Image: …]` patterns), **citations** (bracket numbers, `[^footnotes]`, parenthetical author–year), **technical tokens** (UUID, `0x` hex, long hashes, Windows/UNC/Unix paths, emails), **document chrome** (“page *n* of *m*”, figure/table line prefixes, dot-heavy TOC lines), and **emoji** (broad Unicode ranges). ## 0.3.5

- **Speak preprocessing** ([`narrator/speak_preprocess.py`](narrator/speak_preprocess.py)): optional removal of **hyperlinks** (markdown `[text](url)`, bare URLs, `www.`, `mailto:`) and **math** (`$$…$$`, `\[…\]`, `\(…\)`, common `\begin{equation}`-style blocks, inline `$…$` except simple currency-like `$3.50`, and Unicode mathematical alphanumeric symbols).
